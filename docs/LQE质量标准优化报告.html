0执行摘要

本地化质量评估（LQE）的国际共识，是用"错误类型学（按维度分类问题）+ 严重度分级 + 按字数归一"的方法，对译文质量打出可量化、可对比的分数1,10。 现有评估体系方向与之一致，本次调研发现其与国际标准存在 8 项差距（3 项为高优先级），并已完成 7 项优化，全部平稳兼容——不改变既有评分口径。

本次最关键的一项优化：数值一致性校验。 游戏里的伤害、数量、等级、概率等数字一旦在翻译中被改动（例如 100 被误译成 1000），会直接影响玩法平衡，属最高危缺陷。 原体系只检查数字"格式"（如千位分隔符），完全不比对数值本身是否一致。国际主流 QA 工具均把"原文↔译文数字一致"列为标配检查37,38，本次已补齐这一缺口。

1行业质量标准调研综述

本地化质量评估已形成全球通行的方法论内核：把译文问题归入标准化的错误维度，按 Neutral / Minor / Major / Critical 四级严重度赋分，再用加权惩罚 ÷ 字数（统一折算到"每千词"）得出可横向比较的质量分2,3；对致命问题设"一票否决"硬门5。五大框架对照如下。

框架维护方 / 定位顶层维度严重度分级评分 / 阈值来源
MQM
多维质量度量		MQM Council，已并入 ISO 50607 个一级维度 0 / 1 / 5 / 25（指数间距）(1−惩罚÷字数)×100，按每千词归一；通过线可校准 1,2,3,4
ISO 5060:2024

LISA QA

几条对游戏本地化最有借鉴价值的方法论：①"每千词归一"使长短不一的文本可横向比较，是各框架共享的内核，memoQ、Smartling 等主流工具均默认按此输出标准化分数40,18；②极短文本（如单条按钮，<约 250 词）应改用规则判定而非线性打分，否则单个错误就会让分数失真17；③同一问题只归入唯一维度，避免重复扣分1。

3本次优化（已完成）

本轮聚焦高价值、零回归项，共 7 项。每项说明：改了哪里 · 解决什么本地化质量问题 · 为什么这样改（对齐哪个标准）。

优化点改了哪里（做什么）为什么这样改（对齐标准 · 来源）优先级状态
① 评估维度对齐国际七维 把原先 6 个自定义分组，重整为国际标准的七个一级维度（术语、准确性、语言规范、风格、地区惯例、受众适配性、版式与标记）。每条质量问题都能精确归入标准维度。 与 MQM-Core1 / ISO 5060:20245 完全一致（两者共用同一套七维）。便于与客户、第三方用统一语言沟通质量，也利于跨项目横向对比。 P0✓ 完成
② 补齐"受众适配性"维度 新增"受众适配"问题类别，并修复了一处令该维度失效的分类映射。用于标记"译文准确、但不符合目标受众语域或世界观期待"的情况（如把仙侠敬语译成现代口语）。 MQM-Core 第六维独立成维1；DQF 调和模型将"与世界观矛盾文本"列为独立类别16。这是 RPG / MMO 叙事本地化的核心质量维度，原体系无法量化。 P1✓ 完成
③ 动态变量与标记的顺序校验 对游戏文本中的动态变量（玩家名、数值等占位符）与颜色 / 富文本标记，不再只比"数量"，还校验"顺序与配对完整性"。 翻译时变量顺序错位或标记未闭合，会导致游戏内显示错乱甚至崩溃，属功能性严重缺陷。Xbench、memoQ 等均将其列为核心自动检查37,39。 P0✓ 完成
④ 界面字段长度上限校验 可读取文案表中的"长度上限"列，对超出界面字段宽度的译文直接预警；无该列时回退到比例估算。 中译英普遍文本膨胀，固定宽度的按钮 / 菜单极易截断或溢出26。对应 DQF 的"版式—长度 / 截断"维度14，比粗略比例估算更贴近真实显示。 P0✓ 完成
⑤ 排版规范校验 新增对行首尾多余空格、双空格、以及英文译文里误用的全角标点的检查。 这是本地化交付中最高频的低级瑕疵；Verifika 专设"标点与空格"检查类、Xbench、Phrase 均内置相应检查38,37,41，且与项目"全部半角"风格规范一致。 P0✓ 完成
⑥ 数值一致性校验
（本次最关键）		 自动比对原文与译文中的数字，一旦出现漏译或改值（如伤害 100 译成 1000），立即按严重缺陷预警。仅在原文含数字时触发，已处理千位分隔符等格式差异，不误报。 战斗 / 技能数值译错直接影响玩法平衡，是游戏功能性最高危缺陷。各主流 QA 工具均把"数字一致性"列为标配37,38,42，而原体系只查数字格式、不校验数值本身——这是诊断中标为最高漏检风险的一项。 P0✓ 完成
⑦ 评分机制对齐国际惯例 提供两项可选能力：一是"致命错误一票否决"——任何 Critical 级问题无论总分高低直接判不通过；二是输出"每千词错误密度"指标，便于不同长度文本横向比较。两者默认关闭，不改变现有评分口径。 致命错误一票否决是 MQM / ISO 5060 / LISA 的一致硬规则5,3,10；每千词归一是各框架共享的可比性内核，memoQ 质量报告默认按此输出2,40。设为可选，保证平稳过渡。 P0✓ 完成
一个专业判断：不臆造数值。 对优化②⑦ 之外的分级阈值与权重（如不同内容类型该用多少分通过、各维度权重倍数、重复错误折扣比例）， 权威来源只给出方向、不给具体数值14。因此本次不写入任何主观假设的数字，留待用真实历史评估数据校准后再落地，避免把拍脑袋的标准固化下来。

4后续规划

近期应做

  • 按内容类型分桶设阈值：对界面 / 道具说明 / 剧情对话 / 营销文案分别设定质量通过线与维度侧重——即 TAUS DQF 的内容画像方法14,13。具体数值需用真实历史数据标定。
  • 极短文本规则判定：对单条按钮等过短文本改用规则判断，而非线性打分17。
  • 全局一致性校验：同一原文在整个文件内是否始终译法一致37。
  • 复数 / 性别形态校验：多语言语法形态的完整性24,25。

中期可选

  • 重复错误去重计分：避免同一系统性错误在长文件中被反复扣分8；折扣比例待业务确认。
  • 扩展未翻译检测：英文原样复制、空译等情形37。
  • 锁定段改动检测：核验不可译内容（变量名、商标等）未被改写28。
  • 明确职责边界：文本质量评估与功能 / 视觉 / 合规测试的交接界面20。

5结论

现有评估体系的设计方向与国际标准一致，本次以零回归方式补齐了最高价值的部分：评估维度对齐国际七维、五项游戏特有的自动质量检查、以及致命错误一票否决机制。 其中数值一致性校验直接堵住了游戏功能性最高危的漏检口。 后续多为需真实数据标定的分级阈值与权重，建议在积累一批已评估项目后再校准落地，避免主观假设固化为标准。
