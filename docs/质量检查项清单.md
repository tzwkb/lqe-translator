# 质量检查项清单与评估方法（项目中立）

> LQE 评估的**通用方法论**：确定性检查类别表、17 子类→MQM 映射、通用评估原则、计分与流程规则。
> **本文不含任何项目专有案例/术语**。某项目的实判案例、强制译名、拼音保留名单、历史报告分析见该项目 `projects/<game>/<track>/` 下的 `adjudications.md` 与 `lqa_notes.md`。效力顺序：客户裁决 > SG > 本清单方法论。

## 一、确定性检查项（pre-check）

内置检查（键名对应 `checks.json` 的 builtin 开关）+ 语言/PM 反馈新增项，共 23 项。类别与严重度固定如下；适用性由目标语言属性自动开关（见 `target_languages/`，如非拉丁关 N8/#7、无句号体系关 N5、无空格分词关 N7）。

| 键 | 检查内容 | 类别（严重度） | 评估关注点 |
|---|---|---|---|
| untranslated_cjk | 译文残留中文 | Untranslated（Major） | 项目指令允许保留的内容除外 |
| empty_target | 译文为空 | Untranslated（Major） | 与词数统计方式联动（source-chars 模式下仍计分） |
| em_dash | 破折号 — | Punctuation（Minor） | 可按项目配置关闭 |
| color_tags | 颜色标签 #G/#C/#Y…#E 配对 | Markup（Major） | 相对位置须一致，不限于数量 |
| variables | 占位符 {}/%s 缺失或多余 | Markup（Major） | 缺一个占位符常被人工评为 Mistranslation Critical |
| pos_placeholder | %s/%d 顺序错位 | Markup（Major） | 命名式或带索引的占位符允许重排 |
| newline_count | 换行符数量不一致 | Markup（Major） | |
| numbers_consistency | 数值缺失或改动 | Mistranslation（Major） | 仅源含阿拉伯数字时触发；中文数字不误报；上下文确认可移除 |
| length | 超出长度限制 | Length（Major） | 优先 max-length 列；无则按 1.5× 源文长度回退（仅非中日韩源） |
| locale_numbers | 千位分隔符缺失 | Locale convention（Minor） | |
| whitespace | 首尾空格、连续空格 | Punctuation（Minor） | |
| fullwidth_punct | 译文中的全角标点（含弯引号 “”‘’） | Punctuation（Minor） | CJK 目标语言由属性关闭 |
| terminology | 术语表命中但译文未采用 | Terminology（Major） | 锁定术语（profile.lock_statuses）直接判错不可翻案；其余结合语境判；通用词命中≠错误 |
| N1 拼音残留 | 连续拼音音节构成的专名，不在保留名单内 | Mistranslation（Critical） | 半确定性：正则初筛 + 人工/AI 复核；**保留名单见项目 adjudications/lqa_notes** |
| N2 同源异译 | 同文件相同源文对应不同译文（及异源同译 ≥20 字） | Inconsistency（Minor） | |
| N3 序号罗马数字 | 序号/卷号用 Unicode 罗马数字（Ⅰ Ⅱ Ⅲ）非 ASCII | Company style（Minor） | 仅带 Vol/Chapter 等前缀的可确定性判，裸序号交 lens 语义判 |
| N5 句尾终止标点不对齐 | 源有译无 / 源无译加句号 | Punctuation（Minor） | 无句号体系语言自动关闭 |
| N6 中文数字+量词漏译 | 「中文数字+量词」强模式 | Mistranslation（Major） | 译侧接受阿拉伯/英文/泰文数字与数词 |
| N7 单词连续重复 | the the 类 | Grammar（Minor） | 白名单豁免合法重复；非空格分词语言关闭 |
| N8 词内大小写混乱 | AppLe 类 | Spelling（Minor） | 豁免 TB 词形/Mc/iPhone/PvP 型/标签变量；非拉丁关闭 |
| N9 半角成对标点不完整 | ()、[]、直双引号奇偶 | Punctuation（Minor） | 源平衡而译不平衡才报；撇号豁免 |
| 标签数量对账 | `<…>`、`[…]` 源译数量比对 | Markup（Major） | 项目专属格式用 checks.json custom `count_match` |
| 术语缩写大小写 | TB 全大写缩写（HP/ATK）译文不符 | Company style（Major） | 仅查缩写；非拉丁关闭 |
| 省略号样式混用 | 文件内 … 与 ... 并存 | Inconsistency（Minor） | 少数派段报 |
| N4 重复错误计分 | 同源同译同错仅首次计分 | （计分规则） | 其余标 Repeated 记录不罚分 |

## 二、AI 评估项（17 子类）与通用原则

子类体系、权重（1.5/1.0）、四类强制 Major（Terminology / Untranslated / Markup / Length）、严重度分值（Neutral 0 / Minor 1 / Major 5 / Critical 10）见 SKILL.md 与 `_common.md`。17 子类归入 MQM-Core / ISO 5060:2024 七个一级维度（Terminology / Accuracy / Linguistic Conventions / Style / Locale Conventions / Audience Appropriateness / Design and Markup，+ Other）。

**通用评估原则**（评估每条错误前先核）：

1. **三点法则**：译文同时满足准确（核心含义保留）、合规（符合 SG / 项目指令 / 术语表）、合乎语法三项，即视为正确；偏好性改写不计错，至多记 Neutral 建议。
2. **单一归属与从重判定**：每条错误只归一个类别，取最具体一类；严重度在 Minor 与 Major 间无法确定时判 Major。
3. **Word Choice 判定边界**：用词导致含义错误→Mistranslation；含义正确但有更优用词→Unidiomatic（Minor）或 Neutral。
4. **严重度三级基准**：Minor＝可察觉但不致误导；Major＝很可能误导玩家或损害可信度；Critical＝可能引起投诉、法律风险、功能破坏或出现在高可见位置。

各子类的具体判别要点见 SKILL.md 评估表 + 各 lens 文件（T/A/G/R）；**项目实判案例、强制译名、拼音保留名单**见项目 `lqa_notes.md` / `adjudications.md`。

## 三、计分与流程规则

| 规则 | 内容 |
|---|---|
| 重复错误 | 重复（同源同译同错）记录 Repeated=YES 但不计罚分，仅首次计分 |
| 阈值分级 | TEP/MTPE 阈值 98；润色/二审 99（`profile.threshold` 可配） |
| 严重度分值 | Neutral 0／Minor 1／Major 5／Critical 10 |
| 评分公式 | 得分 ＝ (1 − Σ加权罚分 ÷ 词数) × 100；词数首轮锁定 |
| 成组文本评估 | 对联、题目、谜题类按组评估：上下联对仗押韵、题答对应须保持；逐句独立直译破坏成组结构 |
| 文本类型判级倾向 | 规则说明类错译普遍判 Critical；界面短句多归 Unidiomatic 或 Company style；诗句宣传语按风格改写要求评估 |
| 报告格式解析 | 修订列可为 #N/A（审校未提供修订稿）；重复列取值 YES/NO |
