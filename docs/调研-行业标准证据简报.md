## 1. 行业主流质量框架类型学对照表

### 1.1 总览对照表

| 框架 | 维护方/标准 | 顶层错误维度数 | 严重度层级 | 严重度权重/惩罚乘数 | 评分归一化 | 通过阈值 |
|---|---|---|---|---|---|---|
| **MQM (Core)** | MQM Council / ISO 5060:2024 | **7** | 4：Neutral/Minor/Major/Critical | **0 / 1 / 5 / 25**（指数间距，默认） | 每 1000 词（RWC=1000） | 原始默认 **99**（DPI=1）；校准后常设 **90 或 95** |
| **ISO 5060:2024** | ISO/TC 37/SC 5 | **7** [CORRECTED] | 3 惩罚级：Critical/Major/Minor | 项目可配置；Critical 自动 Fail | error score 相对字符/词/行数 | error score 对比 error score threshold |
| **LISA QA Model (v3.1)** | LISA（2011 解散） | ~7 语言类 + 非语言层 | 3：Minor/Major/Critical | **1 / 5 / 10**（跨所有类别平权） | 每 1000 词 | 默认 **50 分/1000 词**（0.05/词），按长度成比例 |
| **SAE J2450** (J2450_201608) | SAE（汽车技术文本） | **7** | 2：serious/minor | **按类别**（非平 5/2，见下） | 加权错误和 / 源词数（缺陷密度，越低越好） | 无统一通过线，买方设定 |
| **TAUS DQF** | TAUS（2011 起；2015 与 MQM 调和） | 6 核心 + 2 附加 = **8** | 4：critical/major/minor/neutral | 可配置（行业对权重共识低）；常见 10/5/1/0 | 每 1000 词 | **灵活**，依内容类型/受众/时效；项目创建时设定 |

### 1.2 MQM 详解

**7 个 MQM-Core 顶层维度**：Terminology、Accuracy、Linguistic Conventions、Style、Locale Conventions、Audience Appropriateness、Design and Markup。
- MQM-Core = 上述 7 维的常用子类型子集；MQM-Full = 完整层级（如 Accuracy → Mistranslation → False friend / MT hallucination）。
- 每错误经 Decision Tree 归入**唯一**维度，再赋严重度（防重复计数）。
- Error Type Weight (ETW) 默认 = 1，可调高（如法律内容上调 Accuracy）。

**严重度乘数（默认 4 级，逐字核实）**：Neutral=0、Minor=1、Major=5、Critical=25。Neutral 不计入 APT；任一 Critical 错误（count>0）可触发自动 Fail；推荐指数间距。

**评分公式（MQM 2.0，逐字核实）**：
- APT (Absolute Penalty Total) = Σ_{i,j} ErrorCount(i,j) × SeverityMultiplier(j) × ErrorTypeWeight(i)
- PWPT = APT / EWC（Evaluation Word Count）
- Raw Quality Score = 1 − PWPT（max-1 刻度）或 **100 − (PWPT × 100)**（max-100 刻度）；规范式 = (1 − APT/EWC) × MSV，MSV=100
- NPT (Normed Penalty Total) = (APT × RWC)/EWC = PWPT × RWC，RWC 通常 = 1000（每千词惩罚分）
- CQS (Calibrated Quality Score) = 100 − (NPT × SF)；SF (Scaling Factor) = DPI/APP = (100 − DPT)/Acceptable Penalty Points

**阈值**：原始评分 Defined Passing Interval = 1（100→99），默认通过线 = **99**；校准放大窗口至可读阈值（90/95）。QS ≥ 通过线则 Pass。

**两个独立示例（核查员更正：原稿误拼为一条链）[CORRECTED]**：
- (a) 阈值设定示例：97.2 原始分 = 每 1000 词 28 惩罚分（对比 legal=99.5/5 分）。
- (b) 校准机制示例：原始分 84.4，APP=20，DPI=15 → SF=15/20=0.75；NPT=15.6 × 0.75 = 11.7 → CQS = 100 − 11.7 = **88.3**。

**Multi-Range 抽样区间 [CORRECTED 下界]**：
- 小样本：**< ~250 词（~15–17 句）** → 用统计质量控制 (SQC, ISO 2859-1 风格)，不直接打分。
- 中样本/线性：约 **~250–300 词起至 ~5000 词**（"500" 仅近似，非硬阈值）。
- 大样本：**> ~5000 词** → 需非线性（对数）评分。

### 1.3 ISO 5060:2024 关键更正

**[CORRECTED] ISO 5060 定义 7 个错误类别（非 6 个）**，明确**包含** Design and markup（格式/版式/超链接）作为第七类。原稿"ISO 仅 6 类、Design and Markup 为 MQM 独有"被多个权威来源否定。MQM-Core 与 ISO 5060 共享同一套 7 个一级类别。
- ISO 5060 使用 **3 个惩罚严重度级（Critical/Major/Minor）**（此点正确）；critical 错误无论总分一律 auto-fail。
- error score = 评估惩罚分总和相对于所评估字符/词/行数；quality rating = error score 与适用 error score threshold 及评级系统比较的分类。
- ISO 11669:2024 提供翻译项目规格（specifications），驱动选用哪些 MQM 维度/权重/阈值。

### 1.4 SAE J2450 详解

**7 个类别 + 按类别 serious/minor 权重（逐字核实，非平 5/2）**：

| 类别 | serious | minor |
|---|---|---|
| Wrong Term (WT) | 5 | 2 |
| Syntactic Error (SE) | 4 | 2 |
| Omission/Addition (OM/AD) | 4 | 2 |
| Word Structure/Agreement (WS) | 4 | 2 |
| Misspelling (SP) | 3 | 1 |
| Punctuation Error (PE) | 2 | 1 |
| Miscellaneous Error (ME) | 3 | 1 |

权重上限 = 5（serious WT），非 1–5。[CORRECTED：某低质来源称"8 类、权重 1–5"系错误，OM/AD 为单一加权行。]

**最终指标** = Σ(count × weight) / 源词数（缺陷密度，越低越好，无 100 分制）。仅测客观语言缺陷，**刻意排除 style/可读性/语域**。
**两条一致性元规则（最可迁移）**：(1) 严重度存疑 → 默认 **SERIOUS**；(2) 跨类别 → 取标准列表中**最早**类别（WT 先于 SE 先于 OM…）。
时间线：1997 任务组（GM/Ford/Chrysler）；2001 首发；2005 成正式规范；当前 J2450_201608。

### 1.5 LISA QA Model 详解

**类别**：Mistranslation、Accuracy、Terminology、Language、Style、Country (locale)、Consistency + 工具实现增加 formatting/layout 与 UI/functional 层。
**平权严重度**：Minor=1、Major=5、Critical=10。
**默认阈值** = 50 分/1000 词（0.05/词），**按样本长度成比例**。单个 Critical（10 分）短样本即自动 Fail。

**示例阈值绑定 [CORRECTED：不可混用两套阈值]**：
- 在 **LISA 默认 50/1000 词**下：2000 词 = 100 分预算 → 3 minor + 2 major = 3×1 + 2×5 = 13 分 = 0.65% = **明确 PASS**。
- RWS/SDL 工具示例为**更严买方配置（~4.5/1000 词）**：300 词允许 ~2 分（3 个 minor 即 FAIL）；2000 词允许 ~9 分（13 分则 FAIL）；单 Critical（10 分）立即 Fail。**勿对同一案例同时援引两套阈值。**

### 1.6 TAUS DQF 详解

**动态质量观**：质量需求随内容类型/目的/受众变化，建立于三参数 **UTS = Utility, Time, Sentiment**（与 Dublin City University 合作）。Utility 与 Sentiment 是清晰的内容区分器，Time 较模糊；UI/网站 = 高 utility，audio/video 与 UGC = 较低。

**四种评估类型**：
1. Adequacy（4 点制）：4 Everything / 3 Most / 2 Little / 1 None
2. Fluency（4 点制）：4 Flawless / 3 Good / 2 Disfluent / 1 Incomprehensible
3. Error Typology/Review（分析式标注）
4. Productivity（words/hour）+ TAUS Efficiency Score（words-per-hour 与 edit distance 的复合）

**DQF-MQM 调和（EU QT21，2015 年 6 月）**：6 核心分支（Accuracy、Fluency、Design、Locale convention、Terminology、Style）+ 附加 Verity、Other = **8 个顶层组**。
- **[CORRECTED] 粒度子类型数**：所引 ACL 2015 论文附录仅列 **~34 个**子类型；"~50" 适用于完整 DQF-MQM 标准但应归于 TAUS/二手材料，非该论文附录。
- 4 严重度：critical/major/minor/neutral，权重可配置，归一化每 1000 词。
- DQF 自 MQM 0.9 起为 MQM 完全一致子集，唯 "kudos"（正面加分）例外。

**阈值灵活性（逐字核实）**：pass/fail 阈值依内容类型、终端用户画像、内容时效性而变；不同内容类型/目标语言可用不同阈值；项目创建时设定。路线图：由 content profiling 自动选择严重度与阈值。
**抽样建议**：MT 评估 ≥200 段；母语/近母语领域评估员；周期复审每组 ≥4 名评估员。

## 2. 游戏本地化特有质量维度

### 2.1 游戏 Bug 四分类

| 类别 | 定义 | 典型项 |
|---|---|---|
| **Linguistic** | 语言层缺陷 | 误译、术语错误、语法、语气/语域 |
| **Cosmetic** | 显示/外观缺陷 | 文本截断 (truncation)、UI 溢出 (overflow)、换行、字体/字符编码 |
| **Functional** | 破坏功能/程序 | 占位符/变量损坏、标签破损、热键冲突、技能数值错误、未译串 |
| **Compliance** | 合规/分级 | 文化敏感内容、年龄分级违规、区域审查 |

### 2.2 关键确定性维度（映射至框架）

- **占位符/变量完整性与顺序**：`{0}`、`%s`、`%d` 等数量、类型、**顺序**一致（源↔译）。映射 MQM Design&Markup / DQF Design / LISA UI-functional。**损坏→Critical 自动 Fail。**
- **标签/颜色标记平衡**：富文本标签、颜色标记成对闭合且属性保留。
- **文本膨胀与长度限制**：译文长度对比源文 + 对比字段宽度上限（固定宽 UI 关键）。DQF 明确列 Design→Length、Truncation/text-expansion。
- **复数与性别一致性**：复数形式、性别一致（语法一致性）。映射 J2450 WS / MQM Linguistic Conventions。
- **术语全局一致性**：与 termbase 一致 + 全局用法一致。DQF Terminology（Inconsistent with termbase / Inconsistent use）。游戏中应上调 ETW。
- **语气 (voice)/语域**：DQF Grammatical register（敬语、正式/非正式称呼一致）——RPG/MMO 对话尤关键。
- **文化本地化与年龄分级合规**：DQF Verity（culture-specific reference，与作品世界观矛盾的文本）；LISA Country/culturalization；区域合规。
- **配音/字幕约束**：时长、字符/行限制、对口型约束（配音）。
- **Locked 段**：不可译/受锁段落须原样保留（变量名、代码、商标）。

### 2.3 内容画像分桶（DQF 内容画像落地）

| 画像桶 | 性质 | 重点维度 | 阈值取向 |
|---|---|---|---|
| **UI/utility**（菜单/按钮/系统/错误消息/设置） | 高 utility 低 sentiment | Accuracy、Locale convention（日期/数字/货币/快捷键/格式）、Design（Length/Truncation） | 严格（功能错误破坏产品） |
| **Item/skill tooltip + 战斗数值** | utility + 术语关键 | Accuracy、Terminology、Fluency | 术语严格 |
| **Brand/marketing**（商店/启动页/广告） | 高 sentiment | Fluency、Style（Unidiomatic/Company style）、Verity/文化适配 | 语气近零容忍 |
| **Narrative/dialogue** | 高 sentiment 低 utility | Fluency、Style、Grammatical register、Verity | 常为 transcreation，允许授权偏离，意义等价优先于字面 adequacy |

## 3. 评分与阈值方法学

- **Error density 归一化**：所有框架共享"加权惩罚和 ÷ 词数"内核，**统一归一化至每 1000 词**（MQM RWC=1000、LISA 50/1000、DQF per-1000），使 5 词 UI 串与 200 词任务日志可比。这是异构游戏字符串评分的核心机制。
- **重复错误折扣**：经核实来源未给出具体折扣数值；可迁移机制为 J2450 元规则——**最早类别胜出、严重度存疑取重**——配合 Decision Tree 单维度归属，从源头消除重复计数与跨轮方差（使 LLM 评分确定化）。
- **内容画像分级阈值**：DQF 明示 pass/fail 阈值随内容类型/受众/时效变化，且可按目标语言覆写。落地为按桶配置不同 metric、严重度权重、通过线（UI 严格、tooltip 术语严格、marketing/narrative 流畅度/风格严格）。
- **小样本低置信**：MQM Multi-Range——单 UI 串（<~250 词）应视线性分数为低置信，转而依赖规则/风险检查。
- **Critical 自动 Fail 门**：MQM/ISO 5060/LISA 一致——任一 Critical（损坏占位符、未译串、错误 UI 提示、错误技能数值、攻击性误译）无视总分直接 Fail。
- **校准放大**：MQM 原始窗口窄（DPI=1，阈值 99），经 SF 校准放大为人类可读阈值（90/95）。

## 4. 自动化 QA 工具可执行的确定性检查清单

适用工具：Xbench、Verifika、memoQ QA、Phrase QA。以下为可纯文本/规则判定项：

- **占位符/变量一致性**：源↔译占位符（`{0}`/`%s`/`%d`/`%1$s`）数量、类型、顺序匹配；缺失/多余/损坏 → Critical。
- **标签/标记完整性**：内联标签、颜色/富文本标记成对闭合，属性保留，顺序一致。
- **术语库一致性**：对照 termbase 检查未用指定术语 / 禁用术语命中。
- **数字一致性**：源↔译数值、范围、单位一致（战斗数值/技能数值关键）。
- **长度/膨胀检查**：译文长度对比源文比例 + 对比字段宽度上限，标记潜在 truncation/overflow。
- **未译/漏译检测**：目标=源（未译）、空译、TM 不当精确匹配。
- **一致性检查**：相同源不同译（inconsistent translation）、相同译不同源。
- **Locale convention 检查**：日期/数字/货币/度量/电话/快捷键格式符合目标区域（UI/系统串客观可自动化）。
- **拼写/标点**：拼写错误、标点错误、双空格、首尾空格、大小写。
- **重复词/字符**：连续重复单词、悬空标点。
- **Locked 段校验**：不可译段原样保留，变量名/代码/商标未被改动。
