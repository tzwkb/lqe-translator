# LQE Excel 修改差异标红设计

## 目标

在后续生成的 `*_lqe.xlsx` 中，将原译与建议译文之间的实际修改范围显示为 Excel 富文本：

- 原译中被删除或替换的内容使用红色删除线。
- 建议译文中新增或替换的内容使用红色字体。
- 未改变内容保持普通黑色字体。

该功能帮助审校人员快速识别修改位置，不改变建议译文内容、评分、修改应用规则或交付文件结构。

## 范围

标红适用于：

1. 单任务 `*_lqe.xlsx` 的 `LQA Scorecard` 逐条错误明细。
2. 单任务 `*_lqe.xlsx` 的 `LQE Results` 逐段结果。
3. 多工作表聚合生成的 `*_lqe.xlsx` 中各子表的 `LQE Results`。

不适用于：

- `*_corrected.xlsx`。
- CSV、TSV corrected 文件。
- 原始输入文件。
- 历史已生成报告的自动改写。

## 显示规则

设原译为 `original`，建议译文为 `suggested`：

- `equal`：两侧均保留普通文本。
- `delete`：仅原译侧显示红色删除线。
- `insert`：仅建议译文侧显示红色字体。
- `replace`：原译侧显示红色删除线，建议译文侧显示红色字体。

使用颜色 `#FF0000`。现有错误行橙色底色、已保护或建议行绿色底色保持不变。

以下情况不创建富文本：

- `corrected` 为 `null` 或空字符串。
- 原译与建议译文相同。
- 句段受保护，因而不显示建议译文。
- 只有需要人工确认的问题且没有程序生成的安全修改。

纯删除修改在建议译文侧可能没有红色字符；删除内容会在原译侧以红色删除线完整显示。

## 差异算法

新增 `scripts/lqe_excel_diff.py`，集中负责差异计算和 Excel 富文本构建。公开接口为：

```python
build_rich_diff(original: str, suggested: str) -> tuple[str | CellRichText, str | CellRichText]
```

实现先使用 `regex` 的 `\X` 将文本拆成 Unicode 扩展字素簇，避免组合字符、泰文音调、旗帜、Hangul Jamo 和 emoji ZWJ 序列跨富文本 run。随后使用 `difflib.SequenceMatcher(autojunk=False)`：

1. 先剥离公共字素前缀和后缀，缩小匹配范围。
2. 常规中段获取 `equal/delete/insert/replace` opcodes。
3. 中段乘积过大时改用最大编辑距离受限的 Myers 差异；超出界限则把整个中段视为替换，避免重复文本二次退化。
4. 分别构造原译侧与建议译文侧运行片段。
5. 合并相邻且格式相同的片段，减少 OOXML 运行数量。
6. 仅在存在实际差异时返回 `CellRichText`；否则返回普通字符串。

选择文本差异而非依赖 `errors[].edit` 坐标，原因是报告始终拥有完整的原译与程序验证后的建议译文，而历史结果、聚合结果或回退路径不一定保留完整坐标。`autojunk=False` 避免长文本或重复字符被启发式忽略。

## Excel 富文本

使用 openpyxl 3.1 提供的：

- `CellRichText`
- `TextBlock`
- `InlineFont`

格式定义：

- 原译差异：`InlineFont(color="FFFF0000", strike=True)`。
- 建议译文差异：`InlineFont(color="FFFF0000")`。
- 相同文本：普通字符串运行，不显式覆盖工作簿默认字体。

最低依赖明确为 `openpyxl>=3.1`，并使用 `regex` 提供完整 Unicode 字素边界。富文本字符串以 Excel 内联字符串写入；即使内容以 `=` 开头，也不得被解释为公式。任一侧超过 Excel 单元格 32767 字符上限时不创建富文本，由现有普通文本写入路径执行 Excel 兼容截断。

## 报告接入

### `LQA Scorecard`

逐条错误明细仍保留现有列和重复错误行。若该段存在非空 `corrected`，第 4 列“原译”和第 5 列“建议译文”使用同一组差异结果；其余列与样式不变。

### `LQE Results`

根据当前动态表头定位“原译”列和“建议译文”列。只有非空建议译文才生成双列富文本。受保护段、无需修改段和仅提醒段保持现有普通文本行为。

### 多工作表聚合

聚合报告复制各子任务结果及现有单元格样式、富文本、显式行列尺寸和冻结窗格后，根据表头重新计算原译与建议译文的富文本。不得把标红写入聚合 corrected 工作簿。

## 兼容性

- 普通方式 `openpyxl.load_workbook()` 读取时，富文本仍呈现为完整字符串，现有值断言继续有效。
- 需要检查格式的测试使用 `openpyxl.load_workbook(..., rich_text=True)`。
- corrected 导出逻辑不调用差异模块，继续仅写入程序验证后的完整建议译文。
- 公式样文本继续按文本处理。
- 报告列名、列顺序、评分、处理方式、保护逻辑和行底色不变。

## 测试

先新增失败测试，再实现功能。测试覆盖：

1. 纯新增：建议译文新增部分为红色。
2. 纯删除：原译删除部分为红色删除线。
3. 替换：原译与建议译文两侧对应部分分别标记。
4. 相同文本：保持普通字符串。
5. 重复文本：`autojunk=False` 下结果稳定且拼接后文本不变。
6. 中文、英文、泰文及组合字符：富文本拼接后与原始完整字符串严格相等。
7. 公式样文本：以字符串保存，不成为公式。
8. `LQA Scorecard` 与 `LQE Results` 均存在预期红色运行。
9. 多工作表聚合报告保留同一差异效果。
10. 无建议、待确认、受保护段不产生差异富文本。
11. corrected Excel 不出现新增的差异富文本或状态样式。

验证命令沿用 `SKILL.md` 的完整测试清单，并把新测试模块加入 `scripts/run_tests.py`。
