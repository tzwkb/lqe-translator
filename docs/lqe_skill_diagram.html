2 边界情况处理 + 人工裁决闭环

核心判据：整词在不在术语库(TB)。在 → 程序按最长匹配自动判；不在 → 缺词/部件冲突走人工门。范围口径（PM 定）：不一致/错误一律列出（含不在 TB 的、TB 占位符、TB 自身错）。

flowchart TD
  T{"术语整词在 TB?"} -->|"在"| LM["按最长匹配取该条 TB
(子词被长词覆盖则不单独追究)"]
  LM --> CMP{"译文 = 该条 TB?"}
  CMP -->|"一致"| OK["判对
(子词与整词译法不一 = 客户库问题, 不报)"]
  CMP -->|"不一致"| TMG["报 Terminology + 给 TB 译法"]
  T -->|"不在"| G{"被含部件情况"}
  G -->|"部件自洽 M6"| CP["用部件译法拼整词 + 标'新词待入库'"]
  G -->|"部件冲突 M7 / 纯缺词 M9"| HM["⟶ 人工门"]
  HM --> RV["待人工裁决表 → PM/客户
回填 terms_*.json(status=Approved) + adjudications.md
下次直接命中、不再问"]
  class OK,LM,CMP,TMG,CP auto
  class HM,RV human

重复与聚合

  • D2 同源同译：去重，只评一次→广播
  • D4 同源异译：报 Inconsistency、要统一
  • D5 聚合后仍分歧：不自动取最严，人工核（防 AI 幻觉错译）

术语库质量

  • T1 占位符/垃圾（音碟吼→ตัวสัตว์）：不采信、列出、提醒客户修库
  • T2/T3 TB 自身错 / 译员更优：列出 + 备注待客户定

口径

  • 范围：不一致/错误都列出
  • severity：表面拼写=Minor
  • 命名(nrc/th)：专名音译优先、避泰服宝可梦官名

3 配置 / 数据 分层架构

三层配置（语言 < 项目共通 < 项目语言 < CLI 显式参数，逐层覆盖）+ job 工作区 + 脚本。

语言层 languages/<code>/

  • attributes.json 语言学事实(script/分词/句末/数字/词数基准)
  • eval_notes.md 语言级评估关注点(泰语性别语尾/敬语…)
  • 已建 en / th

项目共通 projects/<game>/common/

  • adjudications_common.md 源文侧/跨语言/在线主表硬规则
  • NRC 提取物·LQA 模板（数据，gitignore）

项目语言 <game>/<lang>/

  • profile.json 总配置
  • checks.json 内置开关+custom regex
  • adjudications.md 该语言专有裁决
  • terms_*.json 术语(带 status)
  • sg / sources / inputs

job 工作区 jobs/<file>/

  • state.json 跨轮持久
  • sg.txt / terms.json / adjudications.md(共通+语言合并) / lang_notes.md
  • errors*.json / chunks/ / EVAL_SPEC
  • *_lqe.xlsx · *_corrected.xlsx

脚本 scripts/

  • lqe_io.py — read / pre-check / write / apply-fixes / export / lookup-terms
  • lqe_chunk.py — split(去重+覆盖过滤) / merge(广播)
  • lqe_aggregate.py — 聚合 + 分歧待裁决报告
  • finalize_job.sh — 一键 merge→算分→apply-fixes→export
  • lqe_calc.py — 评分(N4 去重 / critical-gate)
  • mastertb_to_terms.py — Master TB → terms(去重/backfill)

4 关键机制 & 优先级

大文件分块（防上下文爆窗）

  • 去重：相同(源,译)只评一次，省 25–29%、杜绝同句异判
  • 覆盖过滤：被长词覆盖的子词不喂 subagent
  • pilot 1 块验证格式 → 再 fan-out
  • merge 校验全 id 覆盖、缺则退回 pre-check

效力 / 合并顺序

  • 裁决效力：CN-EN 更新要求 > Query 裁决 > SG
  • 检查合并：内置默认 < 语言属性推导 < 项目 checks.json < CLI 显式
  • adjudications = 共通 + 语言（read 自动并入）
  • 术语 status：Approved 严格 / WorkingTB 流通 / New 软判

5 评分公式

K类 = Σ severity_points（Neutral 0 / Minor 1 / Major 5 / Critical 10）
L类 = weight × K　（weight：多数 1.5；标点/拼写/locale/other 1.0）
最终得分 = MAX( (1 − ΣL / 固定词数) × 100 , 0 )　阈值 98
词数首轮锁定、迭代不变；N4 同源同译同类同级重复错只计一次

强制 Major：Terminology / Untranslated / Markup / Length（脚本纠正，无论 AI 填值）。
