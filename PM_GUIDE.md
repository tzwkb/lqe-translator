# LQE Translator 项目经理操作手册

> 面向本地化 PM。技术细节见 `SKILL.md`，本手册只讲：它是什么、怎么用、你会被问什么、拿到什么。

---

## 一、它是什么

LQE Translator 是跑在 Claude Code 里的**译文质检 + 自动修正 agent**：

- 按 MQM / ISO 5060 错误体系给译文打分（17 类错误 × 4 级严重度）
- 同时给出每条错误的**修正译文**，自动迭代直到得分 ≥ 98 分
- 已落地项目：洛克王国中→泰（`nrc-th`）、中→英（`nrc-en`）、燕云十六声中→英（`wwm`）

一句话：把译文文件丢给它，拿回一份 LQE 报告 + 一份达标的修正稿。

## 二、你需要准备什么

| 必需 | 说明 |
|------|------|
| 译文 Excel | 含原文列 + 译文列（列名告诉 agent 即可） |

| 可选 | 说明 |
|------|------|
| 风格指南 | docx / xlsx / txt 都行 |
| 术语表 | xlsx / csv / json |
| 项目档案 | **老项目首选**——nrc-th 等已配好 SG/术语/检查规则，一句话调用 |

新语种/新项目第一次跑时把 SG 和术语表给齐；之后 agent 会建好项目档案，复用时只报项目名。

## 三、操作流程

```
你：发起任务（文件 + 项目名）
        │
        ▼
① 初始化 ─── agent 读文件，报告段数/词数/配置加载情况
        │
        ▼
② 术语锁定确认 ─── ★ 可能向你提问（见下节）
        │
        ▼
③ 机器预检 ─── 变量/标签/数字/标点/术语等硬规则，零遗漏
        │
        ▼
④ AI 评估 ─── 语义/风格/语法类错误判定 + 给出修正译文
        │
        ▼
⑤ 评分 ─── 词数固定，按错误扣分
        │
   ≥98 PASS ──→ 输出报告 + 修正稿，结束
        │
   <98 FAIL ──→ 修正写回，输入 /loop 进入下一轮（回到④）
```

### 实际操作只有两步

1. 在 Claude Code 输入 `/lqe-translator`，然后说明任务，例如：

   > 用 nrc-th 项目档案，评估 `~/Desktop/Batch5普老师.xlsx`，原文列"原文"，译文列"泰语"

2. 每轮结束 agent 会报分数；没过就输入 `/loop` 继续，过了就拿文件。

## 四、agent 会问你什么（请提前有数）

**术语锁定确认**（每个项目第一次跑必问）：

- 如果术语表带"状态"列（如 Approved / New），agent 会列出取值问你：**哪些状态算"锁定"？**
- 锁定的含义：该术语判错后不再进入 AI 复核，修正必须采用锁定译法
- 不确定就答"无锁定"——所有术语按语境正常判断；之后客户给了锁定清单再补
- 答案会记进项目档案，**下批不再问**

另外两类无需你操作的保护：

- **RAG/TM 100% 句段**：自动识别、不改动、不扣分
- **客户裁决**：审校回件里的定论会沉淀进项目档案（adjudications），后续批次自动遵守

## 五、产出物

跑完后在 `jobs/<文件名>/` 下：

| 文件 | 用途 |
|------|------|
| `*_lqe.xlsx` | LQE 报告：逐段错误、类别、严重度、修正——**交客户/审校** |
| `*_corrected.xlsx` | 修正稿：原文件结构不变，仅替换译文列——**交付用** |
| `*_lqe_iter{N}.xlsx` | 各轮中间报告（追溯用） |

## 六、评分规则速查

- 公式：`得分 = (1 − Σ加权扣分 ÷ 词数) × 100`，阈值 **98**
- 严重度分值：Minor 1 / Major 5 / Critical 10（Neutral 不扣）
- 准确性类错误权重 1.5，标点格式类权重 1.0
- 词数口径：英语数译文单词；**泰语等无空格语言按源文字符**（已在项目档案配好）
- 强制 Major 项：未翻译、术语、标签变量、超长

## 七、常见问题

**Q：审校回了 QA 反馈文件怎么办？**
发给 agent，说"根据回件优化规则"。误报会被消化成项目规则（裁决/检查项/术语状态），下批自动生效。

**Q：跑一半中断了？**
状态都在 `jobs/<文件名>/state.json`，重新打开会话输入 `/loop` 接着跑。

**Q：能改阈值吗？**
能，项目档案里配置（默认 98）。

**Q：术语表/SG 更新了？**
把新文件给 agent，说明替换哪个项目的配置即可。
